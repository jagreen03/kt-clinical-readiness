# KTS-0000007 Pass 4: Controller Revisions

## Revisions Applied

1. **Endpoint Contract Drift**: Applied Option A. Removed class-level `@RequestMapping("/api")` from `AuthController`. Method-level mappings explicitly defined as `/api/me` and `/auth/logout` to preserve namespace separation within a single controller.
2. **Inbound Header Leak**: Introduced `INBOUND_BLOCK` set (`cookie`, `authorization`) to `ProxyController`. Inbound headers are now filtered against both `HOP_BY_HOP` (RFC 7230 compliance) and `INBOUND_BLOCK` (BFF trust boundary policy) before being forwarded to downstream services.
3. **Response Suppression Scope**: Expanded `SUPPRESS` set to include `server` and `x-powered-by` alongside `content-length`. All three are response-side suppressions serving different sub-concerns (information-leak defense vs. framework content-length correctness) within the same overall defensive posture. 

## Minor Items Disposition

* **Minor 1 (HttpRequestFactory Deprecation)**: Deferred. If `SimpleClientHttpRequestFactory` triggers warnings against Spring Framework 7 during testing, we will swap to `JdkClientHttpRequestFactory` in a future pass.
* **Minor 2 (JSON Envelope Consistency)**: Deferred. Acknowledged the discrepancy between `{"message":"logged_out"}` and `{"error":"..."}` shapes. Normalization is logged for a future cleanup pass.
