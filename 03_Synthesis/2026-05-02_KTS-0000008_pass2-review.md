## Carry-forward Gap 1: Logout return type loses caller signal

Current logout returns void, fire-and-forget. A caller cannot react to logout completion. Change to Observable<void>:

  logout(): Observable<void> {
    return this.http.post<void>(...).pipe(
      finalize(() => {
        this.currentUser.set(null);
        this.router.navigate(['/']);
      })
    );
  }

Same finalize semantics so state clears regardless of caller subscription. Defer to Pass 3 if the error interceptor pass touches the auth surface.

## Carry-forward Gap 2: me() race under concurrent guard activations

Two protected routes activating simultaneously both see currentUser() === undefined, both call me(), both fire requests to /api/me. Wasteful but final state converges correctly. Fix is a shared in-flight observable:

  private inFlight: Observable<User | null> | null = null;

  me(): Observable<User | null> {
    if (this.inFlight) return this.inFlight;
    this.inFlight = this.http.get<User>(...).pipe(
      tap(user => this.currentUser.set(user)),
      catchError(() => {
        this.currentUser.set(null);
        return of(null);
      }),
      finalize(() => { this.inFlight = null; }),
      shareReplay(1)
    );
    return this.inFlight;
  }

Defer to Pass 3 or note as carry-forward. Not a Pass 2 blocker for typical single-protected-route flows.

## Documentation Gap 3: Guard redirect-to-landing UX choice should be in synthesis

Guard redirects unauthenticated users to '/' rather than calling authService.login() directly. The '/' route has the login button (per Cynthia's logout target decision), so the user lands on public, sees the button, clicks, OAuth runs.

Defensible UX. Worth explicitly documenting in synthesis as a UX decision rather than leaving it implicit. Alternative (immediate redirect to Google) is sometimes preferred for SPAs that have no public content. Synthesis should name the choice.
