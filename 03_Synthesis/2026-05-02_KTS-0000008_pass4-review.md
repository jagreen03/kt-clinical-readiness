## Gap 1: Inline styles bypass TriadStack design tokens

Cynthia inline-styled three elements with hex values:

  style="margin-bottom: 1rem; padding: 1rem; background: #f5f5f5; border-radius: 4px;"
  style="background: #eef; padding: 1rem;"
  style="margin-top: 1rem; color: #d32f2f;"

Issues:
- Bypasses TriadStack PGB design tokens established in KTS-0000004.
- Hex values are arbitrary, not referencing the WCAG AA contrast palette.
- Inline styling makes consistent theming impossible.

Resolution: replace inline styles with CSS class names referencing TriadStack tokens. Either via component-scoped styles array using var(--triadstack-*) variables, or by removing inline styles and letting global TriadStack styles cascade.

For demonstrator scope, simplest path: drop inline styles entirely, use semantic class names (.user-state, .success-response, .error-response), let global cascade handle visuals. The component doesn't need pixel-perfect visuals; it needs to demonstrate the round-trip works.

## Gap 2: Subscription not unsubscribed

The fetchSample() method calls .subscribe({...}) without cleanup. If the component is destroyed mid-request, the subscription continues. Modern Angular fix is takeUntilDestroyed:

  import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
  import { DestroyRef } from '@angular/core';

  private destroyRef = inject(DestroyRef);

  fetchSample(): void {
    this.isLoading.set(true);
    this.response.set(undefined);
    this.errorMessage.set('');

    this.proxyService.fetchClinicalSample()
      .pipe(takeUntilDestroyed(this.destroyRef))
      .subscribe({...});
  }

Edge-case memory leak. Fix is one-line additive.
