# KTS-0000008 Pass 3: Error Interceptor Revision

## Revisions Applied

* **Race Condition Mitigation**: Addressed the 401 handling race condition between the global `errorInterceptor` and `AuthService`. Previously, a 401 on `/api/me` or `/auth/logout` would trigger state clearing and navigation in both the interceptor and the service's `catchError`/`finalize` blocks. 
* **URL Exclusion Pattern**: The interceptor now excludes these specific paths using a regex match (`!req.url.match(/\/(api\/me|auth\/logout)/)`). `AuthService` retains complete lifecycle ownership of its own auth-state-determining calls, while the interceptor continues to protect all other downstream proxy and API routes.
