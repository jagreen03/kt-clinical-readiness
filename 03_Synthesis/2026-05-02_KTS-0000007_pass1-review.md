## Critical Gap 1: Spring Boot version is EOL-class outdated

Cynthia pinned 3.2.5. As of May 3, 2026:

- Spring Boot 4.0.5 is the current GA stable, released March 26, 2026.
- Spring Boot 3.5.13 is the latest 3.x patch and is the final 3.x minor.
- Spring Boot 3.5.x OSS support ends June 30, 2026.
- Spring Boot 3.4 OSS support ended December 31, 2025.
- Spring Boot 3.2 is two minor versions past EOL on the open-source track.

For an interview-defense portfolio piece authored today, the right pin is 4.0.5. New projects target 4.0. This is exactly the version-currency check the Pass 1 prompt explicitly asked Cynthia to perform; she missed it.

Resolution: change parent version to 4.0.5.

If there is a deliberate reason to stay on 3.x (compatibility with a legacy library not yet ready for Spring Framework 7), the right pin is 3.5.13, not 3.2.5. Document the reason in the synthesis. Otherwise, default to 4.0.5.

## Critical Gap 2: Missing spring-boot-starter-aop

The Resilience4j Spring Boot 3 starter requires AOP support to power the @CircuitBreaker and @Bulkhead annotations Cynthia plans to use in A-#4. The Resilience4j documentation explicitly lists three required dependencies for annotation-driven use:

  - resilience4j-spring-boot3
  - spring-boot-starter-actuator
  - spring-boot-starter-aop

Cynthia included the first two and omitted the third. Without spring-boot-starter-aop, the application will start without errors and the resilience annotations will be silently ignored at runtime. This is worst-class bug behavior: looks fine, fails open.

Resolution: add spring-boot-starter-aop to the dependencies block.

## Critical Gap 3: Resilience4j 2.2.0 outdated

Latest is 2.4.0 on Maven Central. Update the resilience4j.version property from 2.2.0 to 2.4.0. The 2.4.x line is compiled against Spring Framework 6 and integrates cleanly with Spring Boot 3.5.x. For Spring Boot 4.0 / Spring Framework 7 compatibility, verify against Resilience4j release notes before locking; an upgrade to 2.5.x or beyond may be required if 4.0.5 is the chosen Boot version.

If Pilot chooses Spring Boot 4.0.5 for Gap 1: hold Resilience4j integration until version compatibility is confirmed against Spring Framework 7. The Resilience4j project has open issues tracking Spring Boot 4 / Framework 7 support.

If Pilot chooses Spring Boot 3.5.13 for Gap 1: pin Resilience4j 2.4.0 confidently.

## Minor: mvn test may fail before Pass 3

With spring-boot-starter-security on the classpath and empty application.yml files, the GatewayApplicationTests.contextLoads() smoke test can fail to start the application context. Spring Security applies default lockdown. Worth flagging so the Pilot is not surprised when running mvn test before Pass 3 lands real configuration.

Two options if this becomes a blocker before Pass 3:

1. Add @SpringBootTest(properties = "spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration") to the test class. Strips security autoconfig for the smoke test only.

2. Add a minimal application-test.yml with just enough to satisfy security autoconfig.

Defer until needed; mvn compile is the Pass 1 target and that succeeds with Cynthia is current pom.xml.
