# BFF Endpoints Contract

* GET /auth/login: Triggers 302 redirect to Google's authorization endpoint.
* GET /login/oauth2/code/google: Handles OAuth2 code exchange. This is Spring Security's default callback path, accepted via Pilot decision in A-#1 review (Path B). Spring's OAuth2LoginAuthenticationFilter intercepts automatically; no controller method required.
* POST /auth/logout: Invalidates server-side session and clears JSESSIONID cookie.
* GET /api/me: Returns authenticated user's claims (Name, Email, Roles) to the SPA.
* ANY /api/proxy/**: Spring Cloud Gateway MVC route to forward authenticated requests to downstream microservices, attaching the Bearer token acquired during login.
