# DREAM Elevations: Pipeline Demonstration Records

## Purpose

The Idea Compression Pipeline (Triage → DREAM → Prompt-able →
Prompt-ized → PROMPT) is doctrine documented in 
05_Wiki/Cyclopter/doctrine-and-roster.md. Triage is documented in 
05_Wiki/Cyclopter/triage-protocol.md.

This file documents real-world journeys of specific DREAMs through 
the pipeline. The purpose is twofold:

1. **Audit trail.** Demonstrates that the Triad doctrine is followed 
   in practice, not just declared in theory.
2. **Pattern library.** Future DREAMs can reference how prior elevations 
   were sequenced, what got caught at each gate, and which agent owned 
   which transition.

## Template

For each elevated DREAM, document:

- DREAM source (path to inbox file)
- Triage notes (what Catalyst-001 or Radar caught)
- DREAM → Prompt-able transition (who, when, what was added/scoped)
- Prompt-able → Prompt-ized transition (who, when, what got refined)
- Prompt-ized → PROMPT transition (who, when, what got locked)
- Casey execution outcome (saga record path, files changed)
- Lessons learned (drift caught, doctrine refinements surfaced)

## Cross-Join Protocol

When a new DREAM materially intersects with parked DREAMs in inbox, 
link them explicitly. Cross-joined DREAMs are sequenced together as a 
joined saga rather than independent threads. Cross-references appear 
in the original DREAM file under a "Cross-Joined DREAMs" section.

This protocol surfaced during the Personal Info + MongoDB elevation 
(see Example 01).
